# 🚀 Инструкция по развертыванию Chat AVG Gateway на Windows Server (IIS 10)

В данной инструкции описан процесс настройки профессионального многопользовательского сервера ИИ с использованием Node.js шлюза и IIS в качестве обратного прокси.

---

## 🛠 1. Подготовка окружения

1.  **Node.js:** Установите [Node.js](https://nodejs.org/) (рекомендуется версия 18+).
2.  **Зависимости:** В папке проекта выполните `npm install` (или запустите `install_deps.cmd`).
3.  **Архитектура:** Убедитесь, что папка `llama_cpp` находится в надежном месте (например, `C:\inetpub\ChatAVG`).

---

## 🔐 2. Безопасность и Порты

Система работает на двух портах:
- **8080 (Gateway):** Основной вход для пользователей. Именно этот порт мы будем «пробрасывать» через IIS.
- **8081 (Backend):** Внутренний порт нейросети. Он должен быть закрыт брандмауэром от внешних подключений.

В файлах `start_*.cmd` уже настроен правильный запуск:
- `llama-server.exe` слушает только `127.0.0.1:8081`.
- `node server.js` слушает `0.0.0.0:8080`.

---

## 🌐 3. Настройка IIS (Reverse Proxy)

Для работы IIS в режиме прокси установите:
1.  **URL Rewrite:** [Скачать](https://www.iis.net/downloads/microsoft/url-rewrite)
2.  **Application Request Routing (ARR) 3.0:** [Скачать](https://www.iis.net/downloads/microsoft/application-request-routing)

### Настройка ARR:
1. В **IIS Manager** выберите сервер.
2. Откройте **Application Request Routing Cache**.
3. Нажмите **Server Settings** -> **Enable proxy** -> **Apply**.

### Настройка сайта:
1. Создайте новый сайт в IIS.
2. В разделе **URL Rewrite** добавьте правило **Reverse Proxy**.
3. Укажите адрес: `127.0.0.1:8080`.
4. Теперь сайт будет перенаправлять всех входящих пользователей на Node.js шлюз, где они должны будут пройти авторизацию.

---

## ⚙️ 4. Автозапуск через NSSM

Для работы сервера в фоновом режиме 24/7 без открытых окон терминала:

1. Скачайте [NSSM](https://nssm.cc/download).
2. Запустите командную строку от имени администратора и выполните:
   ```cmd
   nssm install ChatAVGService
   ```
3. В окне настройки:
   - **Path:** `C:\путь\к\проекту\start_8k.cmd` (или другой cmd)
   - **Startup directory:** `C:\путь\к\проекту\`
4. Нажмите **Install service**.
5. Запустите службу через `services.msc`.

---

## 🧪 5. Управление пользователями

После развертывания:
1. Зайдите на ваш домен.
2. Авторизуйтесь под `admin` / `admin`.
3. Перейдите в **Админ-панель**.
4. Создайте аккаунты для ваших сотрудников. Им не нужно знать API-ключи или адреса бэкенда — всё управление (включая лимиты контекста и системные промпты) происходит централизованно.

---

## ⚠️ Решение проблем

- **Ошибка 502.3:** Node.js сервер не запущен или упал. Проверьте статус службы NSSM.
- **Бесконечная загрузка:** Проверьте, что `llama-server.exe` успешно запустился на порту 8081 (это можно увидеть в логах NSSM или запустив cmd вручную).
- **Сброс авторизации:** Убедитесь, что у IIS есть права на запись в папку `data/`, чтобы сервер мог сохранять сессии и настройки.
