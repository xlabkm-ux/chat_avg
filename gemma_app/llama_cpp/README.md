# Инструкция по использованию llama_cpp на Windows Server 2019 / Xeon Ice Lake

Данная сборка `llama_cpp` оптимизирована для работы на современных 64-битных системах Windows с поддержкой расширенных инструкций процессора.

## 1. Системные требования и зависимости
Для работы исполняемых файлов на Windows Server 2019 (или "чистых" облачных образах) необходимо убедиться в наличии следующих компонентов:

*   **Microsoft Visual C++ Redistributable (2015-2022):** Обязателен для запуска.
    *   [Скачать (x64)](https://aka.ms/vs/17/release/vc_redist.x64.exe)
*   **Архитектура:** Только 64-бит (x86_64).

## 2. Оптимизация под Xeon Ice Lake (AVX-512)
Процессоры Xeon Ice Lake поддерживают инструкции **AVX-512**, которые значительно ускоряют вычисления LLM.
*   Сборка содержит специализированный файл `ggml-cpu-icelake.dll`.
*   Программа автоматически загрузит его при обнаружении подходящего процессора.
*   **Проверка:** При запуске в консоли должна появиться строка:
    `load_backend: loaded CPU backend from ...\ggml-cpu-icelake.dll`

## 3. Рекомендованные настройки для облачной инфраструктуры

### NUMA (Non-Uniform Memory Access)
Для серверных процессоров Xeon обязательно используйте флаг `--numa` при запуске `llama-server.exe` или `llama-cli.exe`. Это оптимизирует работу с оперативной памятью.

### Управление потоками (Threads)
В облачных виртуальных машинах рекомендуется использовать количество потоков, равное числу **физических ядер**, а не логических (vCPU).
*   Пример для VM с 16 vCPU (8 физических ядер):
    `llama-cli.exe -t 8 ...`

### Блокировка страниц в памяти (Lock Pages in Memory)
Чтобы избежать замедлений из-за сброса весов модели в файл подкачки:
1.  Запустите `gpedit.msc`.
2.  Перейдите: *Конфигурация компьютера -> Конфигурация Windows -> Параметры безопасности -> Локальные политики -> Назначение прав пользователя*.
3.  Найдите **"Блокировка страниц в памяти" (Lock pages in memory)**.
4.  Добавьте своего пользователя и перезагрузите сервер.

## 4. Файлы бэкенда
В директории находятся оптимизированные библиотеки для различных архитектур:
*   `ggml-cpu-icelake.dll` — для Xeon Ice Lake (рекомендуется).
*   `ggml-cpu-zen4.dll` — для AMD EPYC/Ryzen 7000+.
*   `ggml-cpu-haswell.dll` — универсальный вариант (AVX2).
*   `ggml-cpu-alderlake.dll` — для Intel Core 12-го поколения и выше.

Программа сама выберет наиболее подходящий бэкенд для вашей системы.
