# MIGRATION-002: Production Rollout Strategy (RC)

## Обзор
Этот документ описывает стратегию развертывания ChatAVG v2.3 в промышленную эксплуатацию. Основной фокус на минимизации рисков через постепенную раскатку, теневой режим (Shadow Mode) и канареечные релизы.

## Этапы раскатки (Rollout Phases)

### Этап 1: Shadow Mode (Теневой режим)
- **Цель:** Проверка стабильности и качества V2.3 на реальном трафике без влияния на пользователей.
- **Действия:**
  - Настроить Middleware для дублирования входящих запросов в V2.3 Semantic Layer.
  - Сравнивать ответы V1 и V2.3 в логах Trace Bus.
  - Мониторить задержку (latency overhead) и стоимость (token cost).
- **Критерий перехода:** Стабильная работа Semantic Layer в течение 48 часов без падений.

### Этап 2: Canary Release (5-10% трафика)
- **Цель:** Проверка реального взаимодействия пользователей с новыми функциями.
- **Действия:**
  - Включить Feature Flags (`SEMANTIC_LAYER_ENABLED`, `AGENT_RUNS_ENABLED`) для ограниченной группы пользователей (например, `internal-beta` или по хэшу ID).
  - Сбор фидбека через "Conflict Cards" и оценку артефактов.
- **Критерий перехода:** Отсутствие критических инцидентов (SEV-1/SEV-2) и позитивный фидбек по качеству Semantic Layer.

### Этап 3: Линейное расширение (25% -> 50% -> 100%)
- **Цель:** Масштабирование на всю аудиторию.
- **Действия:**
  - Постепенное увеличение доли трафика.
  - Масштабирование Temporal Workers и LiteLLM инстансов.
  - Мониторинг нагрузки на SQLite (Write-Ahead Log performance).

## Процедуры отката (Rollback Scenarios)

### Сценарий A: Ошибка в Semantic Layer
- **Симптомы:** Ложноположительные блокировки адекватности, высокая задержка.
- **Действие:** Установить `SEMANTIC_LAYER_ENABLED=false`.
- **Время отката:** < 30 секунд (через изменение .env и перезапуск).

### Сценарий B: Критическая ошибка Temporal/AgentRuns
- **Симптомы:** "Зависание" миссий, ошибки в `Temporal UI`.
- **Действие:**
  - Установить `AGENT_RUNS_ENABLED=false`.
  - Отключить Temporal Worker.
- **Время отката:** < 1 минуты.

### Сценарий C: Перегрузка Tool Gateway / Sandbox
- **Симптомы:** Таймауты при вызове инструментов.
- **Действие:**
  - Ограничить Rate Limits в Tool Gateway.
  - Переключить Sandbox на `local` адаптер вместо `E2B` (если применимо).

## План резервного копирования и восстановления
- **База данных:** Ежедневный бэкап `database.sqlite` перед каждым этапом раскатки.
- **Стейт Temporal:** Бэкап PostgreSQL/MySQL (если используется для Temporal Persistence) или экспорт активных воркфлоу.
- **Конфигурация:** Все версии `.env` и `package-lock.json` фиксируются в Git-тегах релиза.
