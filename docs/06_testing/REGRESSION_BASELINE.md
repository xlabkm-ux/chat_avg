# Базовый уровень регрессии (Regression Baseline)

**Версия:** 1.0 (Статус на момент старта v2.3 - Sprint 0)  
**Дата фиксации:** 7 мая 2026

Этот документ фиксирует текущее (рабочее) состояние системы V1. Любые изменения в архитектуре при переходе на V2.3 **не должны сломать** этот базовый функционал (особенно на "Fast Path").

## 1. Базовый функционал (Must-Have)
Следующие сценарии работают сейчас и должны работать после любых рефакторингов:
1. **Аутентификация:** Пользователь может залогиниться под учетной записью администратора.
2. **Управление провайдерами:** В Admin UI можно сохранить ключи (OpenAI, DeepSeek, Qwen) и они применяются.
3. **Простой чат (Fast Path):** Пользователь может отправить текстовое сообщение провайдеру и получить потоковый (streaming) ответ без ошибок 500.
4. **Хранение сессий:** История чата сохраняется в базе данных (SQLite) и корректно загружается при обновлении страницы.
5. **UI:** Статические файлы раздаются корректно, нет ошибок CORS.

## 2. Базовые метрики производительности (Latency)
*(Примерные значения V1 для бенчмарка)*
*   **Time to First Token (TTFT) для простого чата:** ~0.8 - 1.5 сек (в зависимости от провайдера).
*   **Загрузка интерфейса:** < 0.5 сек.

*Цель для v2.3 Fast Path:* Сохранить TTFT < 1.5 сек. Внедрение Gateways и Temporal не должно замедлять простой чат.

## 3. Известные проблемы V1 (Будут исправлены в V2.3)
*   Ошибки Gateway/MCP proxy при нестандартных ответах провайдеров.
*   Отсутствие строгой изоляции при выполнении кода (риск выполнения на хосте).
*   Зависание RAG (Grok Collections) при обработке больших результатов.
