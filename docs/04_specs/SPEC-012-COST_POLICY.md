# SPEC-012: Cost Policy and Run Estimate

**ID:** SPEC-012 | **Версия:** 1.0 | **Статус:** Active  
**Владелец:** Backend + SRE | **Обновлено:** 2026-05-07

---

## 1. Назначение (Purpose)

Контроль стоимости (Cost Policy) обеспечивает предсказуемость расходов на использование AI-моделей и инфраструктуры (например, Sandbox). Каждый AgentRun должен иметь оценку стоимости (Estimate) и не превышать установленные лимиты.

## 2. AgentRun Estimate

Перед запуском или во время планирования миссии агент (или платформа) рассчитывает максимальный бюджет для текущего `AgentRun`:

```typescript
interface CostEstimate {
  estimatedTokens: number;
  maxCostUsd: number;
  currency: 'USD';
}
```

## 3. Отслеживание (Cost Tracking)

В течение выполнения миссии:
1. Вызовы LLM-моделей (Model Gateway) генерируют события потребления.
2. Использование песочницы тарифицируется посекундно/поминутно.
3. Генерируются события `cost.committed` (Trace Events), которые агрегируются в `run.service`.

## 4. Policy Enforcement (Применение лимитов)

Если накопленная стоимость превышает `maxCostUsd`:
- PolicyEngine возвращает `deny` с причиной "Cost limit exceeded".
- `AgentRun` переходит в состояние `failed` или `requires_action` (если система настроена запросить у пользователя увеличение бюджета).

## 5. Audit Logging

Каждый факт генерации коста должен записываться в `AuditService` с указанием провайдера, модели и количества токенов (Prompt / Completion) для последующей аналитики и биллинга.
