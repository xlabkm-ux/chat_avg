# План выполнения: Sprint 17 — Release Candidate

**Цель:** Закрыть все релизные гейты (Architecture, Security, Performance, Semantic, Migration) и подготовить систему ChatAVG v2.3 к промышленному запуску.

## Шаг 1: Полное регрессионное тестирование
- **Задача:** Выполнить все существующие тесты и сравнить результаты с базовой линией.
- **Детали:**
  - Запустить `npm run test:release` в `cons/chatavg`.
  - Убедиться в 100% прохождении тестов (unit, contract, security, integration, sandbox).
- **Ожидаемый результат:** Отчет о регрессии без ошибок.

## Шаг 2: Глубокий аудит безопасности (Red-Team)
- **Задача:** Провести комплексную проверку безопасности.
- **Детали:**
  - Тестирование Prompt Injection (попытки обхода системных промптов).
  - Проверка эскалации привилегий инструментов (вызов write/exec инструментов без прав).
  - Проверка изоляции арендаторов (cross-tenant leakage).
  - Аудит Egress Policy (попытки выхода песочницы во внешнюю сеть мимо белых списков).
- **Ожидаемый результат:** `docs/07_security/SECURITY_SIGN_OFF_RC.md`.

## Шаг 3: Полная оценка качества ИИ (Evaluation)
- **Задача:** Провести финальные замеры качества на Golden Set.
- **Детали:**
  - Запустить `npm run eval:full` для RAG и Semantic Layer.
  - Целевые показатели: Semantic Accuracy >80%, RAG Faithfulness >90%.
  - Проверка сценариев отказа (refusal) и восстановления (recovery).
- **Ожидаемый результат:** `docs/06_testing/FULL_EVAL_REPORT_RC.md`.

## Шаг 4: Разработка стратегии миграции и отката
- **Задача:** Подготовить документ `MIGRATION-002`.
- **Детали:**
  - Описать этапы раскатки (Shadow mode, Canary 5%, Full Rollout).
  - Процедуры быстрого отката (Rollback) и восстановления из бэкапа.
- **Ожидаемый результат:** `docs/09_ops/MIGRATION-002-PROD_ROLLOUT.md`.

## Шаг 5: Консолидация ранбуков (Operational Readiness)
- **Задача:** Завершить подготовку инструкций по эксплуатации.
- **Детали:**
  - Добавить ранбуки для Temporal, LiteLLM, E2B, Knowledge Gateway, Tool Gateway.
  - Описать действия при резких скачках стоимости (Cost Spike Mitigation).
- **Ожидаемый результат:** Полный набор в `docs/09_runbooks/`.

## Шаг 6: Финальный Sign-off
- **Задача:** Собрать подтверждения готовности от всех стейкхолдеров.
- **Детали:** Архитектура, Бэкенд, Фронтенд, QA, Безопасность, DevOps, Продукт.
- **Ожидаемый результат:** `docs/05_delivery/RELEASE_CANDIDATE_REPORT_RC.md`.
