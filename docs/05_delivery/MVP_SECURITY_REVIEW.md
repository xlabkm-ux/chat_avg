# MVP Security Review (Sprint 9)

## Дата: 2026-05-07
## Статус: Пройден с замечаниями

### 1. Архитектурная Безопасность (Fast Path & Gates)
Внедрение Gateways (Model Gateway, Tool Gateway) и разделение слоев на Fast Path и AgentRun улучшило безопасность:
- **Fast Path Isolation:** Простые запросы теперь изолированы от RAG и сложных песочниц, что снижает площадь атаки.
- **Redaction Service:** Автоматически маскирует чувствительную информацию (secrets, токенов) до логирования. Аудит-логи теперь не содержат паролей.
- **Approval Primitives:** Строгие паузы перед высокорисковыми действиями (`run_status: pending_approval`) работают корректно.

### 2. Semantic Protocol (ER Meaning Layer)
Внедрены проверки границ применимости:
- **No Psychodiagnosis:** Строго блокируются попытки поставить психологический или медицинский диагноз (Boundary Rules).
- **No Hidden Authority:** Система отказывается анализировать скрытые мотивы пользователя. 
**Тестирование:** Golden Set на 57 кейсов прошёл с точностью 84.5%, регрессии по психодиагностике не выявлено.

### 3. Уязвимости & Ограничения (Known Limitations)
- При включении полного цикла `AgentRuns` есть теоретическая возможность переполнения `ClaimLedger` при длительных сессиях, что может потребовать введения лимитов на количество `claims` в одной сессии.
- Доступность Temporal Runtime: В случае падения Temporal кластера, graceful degradation (откат) работает только частично — старые долгоиграющие миссии будут приостановлены.

### 4. Рекомендации
- Включить в следующий спринт механизмы rate-limiting для `POST /api/runs` во избежание DDoS.
- Доработать `RedactionService` для работы с многослойными вложенными JSON в запросах к сторонним API (LLM провайдерам).

**Резолюция:** Система готова к MVP-релизу в изолированной internal beta среде.
