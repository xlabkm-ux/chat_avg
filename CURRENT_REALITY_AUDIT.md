## 1. Детальный аудит (P0/P1 Проблемы)

### 🚨 Критические (P0) — Безопасность и Runtime
- **Sandbox:** `LocalAdapter` используется по умолчанию. Отсутствует жесткий `fail-closed` в проде. Риск выполнения кода на хосте.
- **Durable Runtime (Temporal):** Workflow — демонстрационный, activities — пустые моки. Нет настоящей устойчивости, идемпотентности и защиты от сбоев.
- **Security:** 
    - Потенциальные SSRF через кастомные URL провайдеров.
    - Пароль администратора может попадать в логи.
    - Отсутствие полной идемпотентности для вызовов инструментов и ранов.

### ⚠️ Высокие (P1) — Функциональность
- **Knowledge Gateway (RAG):** Mock-retriever. Реальный поиск (FTS5/Vector) не интегрирован. Цитаты не проверяются -> риск высоких галлюцинаций.
- **Semantic Protocol:** Rule-based PoC, слабая персистентность событий и утверждений (claims).
- **Architecture:** Сильное расхождение между документацией и реализацией.
