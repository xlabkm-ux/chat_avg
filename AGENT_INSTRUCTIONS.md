# 🤖 Инструкции для AI-агента Antigravity

Этот файл содержит обязательные правила поведения агента в данном рабочем пространстве.

## ⚠️ Правило планирования (ОБЯЗАТЕЛЬНО)

**Прежде чем выполнять любые запросы на разработку, создание новых функций или сложную модификацию приложения/модели, агент ОБЯЗАН:**

1.  **Составить краткий план** предлагаемых изменений.
2.  **Сообщить этот план пользователю** в ответном сообщении.
3.  **Дождаться явного утверждения** плана пользователем.
4.  **Приступать к выполнению только после получения подтверждения.**

## 🔄 Итеративный процесс разработки (Спринты)

**При выполнении любого этапа плана агент ОБЯЗАН придерживаться следующего цикла:**

1.  **Дробление**: Реализовывать этап короткими итерациями (спринтами), избегая слишком объемных правок за один раз.
2.  **Завершение спринта**: Каждая итерация должна обязательно заканчиваться цепочкой действий:
    - **Тестирование**: Проверка функционала и отсутствие регрессий.
    - **Обновление карты проекта**: `node dev_studio/refresh.js` — обновить `PROJECT_MAP.md` перед фиксацией.
    - **Коммит**: Фиксация изменений в Git с информативным сообщением (включая обновлённый `PROJECT_MAP.md`).
    - **Пуш**: Отправка кода в удаленный репозиторий для синхронизации.

## 🚫 Запрет на использование Python

**ЗАПРЕЩАЕТСЯ** использовать, писать или предлагать скрипты и серверные компоненты на языке **Python** без явного согласования и разрешения от пользователя. В качестве основного серверного языка следует использовать **Node.js**.

## 📋 Контекст плана и управления проектом

**В начале КАЖДОГО нового чата агент ОБЯЗАН:**

1. Прочитать центральный бэклог проекта:
   `./PROJECT_BACKLOG.md`
2. Прочитать актуальный аудит интерфейса:
   `./UI_AUDIT.md`
3. Прочитать карту архитектуры проекта:
   `./PROJECT_MAP.md`
4. Кратко сообщить пользователю: текущий статус (какой этап в работе, какие критические ошибки остались).
5. **Не требовать от пользователя повторного объяснения контекста** — агент восстанавливает его самостоятельно из этих файлов.
6. **Перед правкой любого файла** — проверить в `PROJECT_MAP.md` зависимости этого файла и убедиться, что изменения не сломают связанные модули.

Рабочая папка проекта: `./cons/llama_cpp/`
WebUI папка: `./cons/chatavg/webui_original/`

## ✅ Чеклист выполненных работ

**После завершения и тестирования каждого этапа плана агент ОБЯЗАН:**

1. Обновить чеклист в файле `PROJECT_BACKLOG.md` — отметить выполненные пункты символом `✅`.
2. Записать краткий итог этапа: что сделано, что изменилось, какие файлы затронуты.
3. Зафиксировать дату выполнения рядом с пунктом.
4. **Не переходить к следующему этапу** без обновления чеклиста текущего.

Формат записи в чеклисте:
```
- [x] ✅ Этап 1 выполнен — 2026-05-01
      Файлы: style.css (полная замена), index.html (мета-теги)
      Итог: mobile-first CSS, iOS safe-area, все inline-styles убраны
```

## 🧪 Обязательное самотестирование перед передачей пользователю

**Перед тем как сообщить пользователю о готовности этапа, агент ОБЯЗАН:**

1. **Запустить браузер** и визуально проверить результат через `browser_subagent`.
2. Проверить минимальный набор сценариев:
   - Страница открывается без ошибок в консоли
   - Базовый сценарий (логин → сообщение → ответ) работает
   - Мобильная верстка корректна (viewport 375px)
   - Нет регрессий в функциональности, которая работала до изменений
3. Зафиксировать результат проверки (скриншот или словесное описание).
4. **Только после успешной самопроверки** сообщить пользователю: «Этап завершён и проверен, готово к вашему тестированию».
5. Если самопроверка выявила ошибки — исправить их самостоятельно до передачи пользователю.
