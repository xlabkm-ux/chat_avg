## 🔴 Критические проблемы (P0 - исправить немедленно)

### 1. **22 пустых документа** создают иллюзию готовности
Наиболее критичные:
- `docs/03_adr/ADR-001-temporal-durable-runtime.md` (ключевое архитектурное решение!)
- `docs/07_security/PROMPT_INJECTION_DEFENSE.md` (security gap!)
- `docs/09_ops/OBSERVABILITY_DASHBOARDS.md` (ops blindness)

**Решение:** Либо заполнить, либо удалить и перенести в backlog.

### 2. **Дубликаты и конфликты именования**
```
❌ Три разных файла с ID "RUNBOOK-003"
❌ Два SPEC-024 в разных папках
```

**Решение:** Стандартизировать нумерацию, объединить дубликаты.

### 3. **Broken Links повсеместно**
- Жестко закодированные пути (`c:\AG\agsys`)
- Ссылки на несуществующие файлы (`CONTRACT_TEST_PLAN.md`)
- Устаревшие ссылки на workdoc/

**Решение:** Заменить на относительные пути, добавить CI link checker.

### 4. **Расхождение docs ↔ код**
PROJECT_BACKLOG заявляет 17+ завершенных спринтов, но:
- Sprint 7 (Temporal) — моки, не production
- Sprint 14 (Sandbox) — LocalAdapter по умолчанию (security risk!)
- RAG Eval Plans — пустые файлы

**Решение:** Честно обновить статусы, добавить disclaimers.

## ⚠️ Высокие приоритеты (P1 - перед RC)

### 5. **Нет единой точки входа**
Новый разработчик теряется в 125 файлах без master index.

**Решение:** Создать `docs/README.md` с навигацией по ролям.

### 6. **Незаполненные ADR**
Критические решения не задокументированы:
- Почему Temporal? (ADR-001 пуст)
- Почему E2B Sandbox? (ADR-003 пуст)
- Почему MCP только для tools? (ADR-004 пуст)

**Решение:** Заполнить все 5 ADR по стандартному шаблону.

### 7. **Почти нет диаграмм**
Только 1 Mermaid-диаграмма на весь проект! Нет визуализации:
- C4 архитектуры
- State machines (хотя есть SPEC-006)
- Sequence diagrams для Approval flow

**Решение:** Добавить 10+ ключевых диаграмм.

### 8. **Фрагментированная структура**
```
docs/04_specs/   ← Спецификации здесь
docs/specs/      ← И здесь! (дублирующая папка вне нумерации)
```

**Решение:** Реструктурировать папки (см. полный отчет).
